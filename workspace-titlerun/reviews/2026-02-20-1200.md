## 🟡 Major Issues (Fix This Sprint)

### M1: pickValueEngineV2 Missing Input Validation on getPickValueV2()
- **Expert:** Security Architect (Sarah Chen)
- **File:** `src/services/pickValueEngineV2.js` (line ~200, function entry)
- **Impact:** Public function `getPickValueV2({ round, slot, year, leagueSize, isSuperflex, leagueContext })` doesn't validate inputs before using them in array lookups. If caller passes `round=999` or `slot=-1`, the function could crash or return undefined instead of gracefully handling invalid input.
- **Fix:** Add input validation at function entry:
  ```javascript
  if (typeof round !== 'number' || round < 1 || round > 4) {
    logger.warn(`Invalid round: ${round}`);
    return 0;
  }
  if (typeof slot !== 'number' || slot < 1 || slot > maxSlot) {
    logger.warn(`Invalid slot: ${slot}`);
    return 0;
  }
  ```
- **Deduction:** -5 (Major Bug)

- **Expert:** Node.js Performance Engineer (Priya Sharma)
- **File:** `src/services/pickCalibrationService.js` (line ~110)
- **Impact:** The calibration service queries `information_schema.columns` on every run to detect the value column name. This is a meta-query that's slower than direct column access. Once deployed, the column name won't change — this should be cached in-memory or configured once.
- **Fix:** Cache the result in a module-level variable with TTL=1 hour, or use an env var `TITLERUN_VALUE_COLUMN=titlerun_value`.
- **Deduction:** -1 (Minor Bug)

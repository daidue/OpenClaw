## 🟡 Major Issues (Fix This Sprint)

### 1. Missing Input Validation on Pick Calibration Service
- **Expert:** Security Architect (Sarah Chen)
- **File:** `src/services/pickCalibrationService.js` (lines 60-95)
- **Impact:** `parsePickText()` accepts raw user input (UTH text values) without sanitization. While UTH is a trusted source, the function handles compound picks with string splitting (`+`, `,`) and regex replacement without bounds checking. If UTH data is ever corrupted or malicious, this could cause regex DoS or injection into downstream queries.
- **Fix:** Add input length limits (max 100 chars), whitelist allowed characters (`[a-zA-Z0-9 +,'-]`), and validate output tier existence before returning.
- **Deduction:** -8 (Major Bug)

### 2. Unhandled Promise Rejection in Daily Scraper UTH Block
- **Expert:** Node.js Performance Engineer (Priya Sharma)
- **File:** `src/services/dailyScraperService.js` (lines 519-538)
- **Impact:** The UTH scraper block catches errors and logs them, but if `uthService.refreshValues()` throws, the error is caught and logged... but the promise chain continues. If UTH scraper fails silently, the pick calibration service (which depends on UTH data) will run with stale data and produce incorrect calibrations.
- **Fix:** Either surface UTH failures to the job orchestrator (throw after logging) OR add explicit staleness checks in `pickCalibrationService` to refuse calibration if UTH data is >7 days old.
- **Deduction:** -8 (Major Bug)

### 3. N+1 Query Pattern in Pick Calibration
- **Expert:** Database Engineer (Marcus Rodriguez)
- **File:** `src/services/pickCalibrationService.js` (line ~110 where query starts)
- **Impact:** Code truncated in review but based on pattern in uthService.js (bulkResolve imports), the calibration likely fetches UTH data row-by-row or joins without proper indexing. If UTH has 500+ players, this could be 500 queries.
- **Fix:** Use a single JOIN between `titlerun_values` and `player_uth_values` (or whatever table stores UTH data). Add composite index on `(player_id, source, updated_at)`.
- **Deduction:** -8 (Major Bug)

### Critical Patterns Learned
- **üî¥ `CI=TRUE` + CRA = build failures** ‚Äî Every unused import or missing hook dep kills the build. Must lint before pushing.
- **üî¥ Cloudflare Pages production branch = `main`** ‚Äî All code must be on `main` to go live. `master` = preview only.
- **üî¥ MailerLite JSONP endpoint returns `success:true` for ANY request** ‚Äî Even empty ones. Not a reliable indicator of actual subscription.
- **üî¥ Hidden iframe = only reliable cross-origin form submission** ‚Äî `no-cors` fetch, JSONP, and MailerLite's own JS all had issues.
- **üî¥ Parallel sub-agent builds create lint debt** ‚Äî 47 warnings accumulated from 8+ agents building simultaneously.
- **üî¥ TypeScript files + CRA + CI=true = build failure** ‚Äî CRA treats .tsx type errors as warnings ‚Üí CI=true kills the build. Must convert to .jsx or fix all type issues.
- **üî¥ Sub-agent audit output can get truncated** ‚Äî 200K token limit means complex audits may not return a summary. Check git log for actual work done.
- **üî¥ Git submodule ghosts persist after .gitmodules deletion** ‚Äî Must `git rm --cached` the 160000-mode entries. Cloudflare fails on "No url found for submodule path". Sub-agents re-add them on every commit. FIXED: Added to `.gitignore` (commit `5488748`).
- **üî¥ Sub-agents re-add submodule entries** ‚Äî Every sub-agent that commits re-stages the 160000 gitlink entries. Must either `git rm --cached` after each sub-agent commit, or block via `.gitignore` (now done).

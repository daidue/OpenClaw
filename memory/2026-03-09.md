## Afternoon: Scoring System Problem Discovery

**Taylor Feedback (1:01 PM):**
> "It successfully deployed. I'm seeing the new 'Why This Trade Works' dropdown which is great, but there's so much information on this page and most of it is broken/not dynamic. For example, all of the 'Needle Movers' are at 75%. 75% of what? (...) All 54's across the board are all confusing."

**Key Issues Identified:**
1. All needle mover scores show 75% (appears hardcoded)
2. All trade overall scores show 54 (no variation)
3. All fairness scores show 51 (no variation)
4. Unclear what metrics mean (acceptance? mutual benefit? overall?)
5. Information overload (too many unexplained metrics)

### Expert Panel Analysis (Opus)

**Agent:** `trade-scoring-overhaul` (4-hour comprehensive review)

**Finding:** Scores are NOT hardcoded — they're **mathematically convergent**

**Root Causes:**
1. `calculateLineupImprovement` only measures value lost as % of total roster value
   - For a 50K roster, 500-value net gain = 1% improvement
   - Maps to score ~45 on myImprovement scale
   - ALL trades cluster at 40-55 range

2. Acceptance baseline of 50 with narrow ±5 to ±15 factors
   - 85% of trades score 40-65 on acceptance
   - Very narrow variation

3. Needle Movers = top-3 by myImprovement score
   - Since all myImprovement scores ~45, they appear identical at 75%

**Files Analyzed:**
- `titlerun-api/src/services/tradeFinderService.js` (scoring logic)
- `titlerun-api/src/services/acceptancePredictionService.js` (acceptance prediction)
- `titlerun-api/src/services/tradeAnalysisService.js` (trade analysis)

### Recommended Fixes (Priority Order)

| Fix | Time | Impact |
|-----|------|--------|
| Replace value math with lineup simulation (`getOptimalLineup` pre/post) | 2h | Fixes myImprovement clustering |
| Lower acceptance baseline to 35, widen factor ranges | 1.5h | Fixes acceptance clustering |
| Non-linear fairness curve | 0.5h | Fixes fairness clustering |
| Quality-gate Needle Movers (≥65 overall, ≥40 acceptance) | 0.5h | Fixes "all 75%" display |

**Minimum Viable Fix:** 4.5 hours (will create natural score variation)
**Full UX Overhaul:** ~14 hours (includes progressive disclosure, new labels, dynasty outlook metric)

**Deliverable:** `TRADE_SCORING_OVERHAUL.md` (36KB comprehensive report)

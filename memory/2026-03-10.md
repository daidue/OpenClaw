**Fix #1: Position Grading Algorithm**
- **File:** `titlerun-api/src/services/tradeFinderService.js` line 172
- **Problem:** Old code compared to league *average* using ratio (everyone clustered near 1.0 → all F's)
- **Solution:** Percentile ranking — counts how many teams you're better than
- **Logic:** Top 20% = A, Top 40% = B, Middle = C, Bottom 40% = D, Bottom 20% = F
- **Code:**
  ```javascript
  const betterThanCount = teams.filter(t => myStarters > t.starters).length;
  const percentile = betterThanCount / (leagueSize - 1);
  // A+ if best, A if top 15%, B+ if top 30%, etc.
  ```

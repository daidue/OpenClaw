**Fix #1: Position Grading Algorithm**
- **File:** `titlerun-api/src/services/tradeFinderService.js` line 172
- **Problem:** Old code compared to league *average* using ratio (everyone clustered near 1.0 → all F's)
- **Solution:** Percentile ranking — counts how many teams you're better than
- **Logic:** Top 20% = A, Top 40% = B, Middle = C, Bottom 40% = D, Bottom 20% = F
- **Code:**
  ```javascript
  const betterThanCount = teams.filter(t => myStarters > t.starters).length;
  const percentile = betterThanCount / (leagueSize - 1);
  // A+ if best, A if top 15%, B+ if top 30%, etc.
  ```

## Lessons Learned

1. **Adversarial audits are worth it** — 7-minute audit caught 3 production-breaking bugs. Would have caused user-facing failures.
2. **Fix bugs before deploy, not after** — Tempting to ship and "fix later," but P0 bugs are non-negotiable blockers.
3. **Placeholder strategies work** — Beta banner + graceful fallback lets us ship UI now, iterate on algorithm accuracy later.
4. **Parallel build + sequential audit** — Build team parallelized work, audit team found issues serially. Good balance.
5. **Test failures are red flags** — 1 failing backend test was a P0 bug. Never deploy with failing tests.

## Flush 17:30 EST ‚Äî Pull-to-Refresh Bug + Code Review

### Taylor Bug Report: Dashboard auto-refreshing rapidly on mobile
- **Symptom:** Page auto-refreshing at very high rate on mobile
- **Root Cause:** `usePullToRefresh` hook had `pullDistance` (state) and `onRefresh` (unstable function ref) as `useEffect` dependencies
  - Every touch event ‚Üí `setPullDistance` ‚Üí re-render ‚Üí effect re-runs ‚Üí listener churn
  - `handleRefresh` created inline without `useCallback` ‚Üí new ref every render ‚Üí effect re-runs
  - iOS overscroll bounce at top of page could repeatedly exceed `threshold` ‚Üí multiple `refreshAll()` calls
- **Fix (commit `2dc0731`):**
  1. Rewrote hook to use refs (`pullDistanceRef`, `onRefreshRef`, `isRefreshingRef`) instead of state in useEffect deps
  2. Effect now runs once on mount, handlers read current values via refs
  3. Added 3-second cooldown (`lastRefreshRef`) between refreshes
  4. Wrapped `handleRefresh` in `useCallback` in Home.jsx
- **Deployed:** Cloudflare Pages auto-deploy from main

### Hotfix #3 - connected_leagues Table Missing (11:56 PM - 12:02 AM)
**Error:** `relation "connected_leagues" does not exist`
**Commit:** a04577f

**Root cause:** Migration v2026021502 tried to ALTER TABLE connected_leagues, but table never existed. Migration 060 creates it later, but runs AFTER v2026021502.

**Fix:** Added CREATE TABLE IF NOT EXISTS before ALTER statements in migration v2026021502:
```sql
CREATE TABLE IF NOT EXISTS connected_leagues (
  league_id VARCHAR(50) PRIMARY KEY,
  user_id VARCHAR(50),
  league_name VARCHAR(255),
  is_active BOOLEAN DEFAULT true,
  connected_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
)
```

**Strategy:** Create table with single-column PK (league_id), Migration 060 later drops and recreates with composite PK (league_id, user_id).

**Deployment:** Pushed 11:59 PM, server recovered 12:02 AM

### Hotfix #4 - CHECK Constraint Syntax Error (12:00 AM - 12:05 AM)
**Error:** `current transaction is aborted, commands ignored until end of transaction block`
**Commit:** 178dac9

**Root cause:** Inline CHECK constraint in ALTER TABLE ADD COLUMN fails in PostgreSQL:
```sql
ALTER TABLE connected_leagues 
ADD COLUMN IF NOT EXISTS league_type VARCHAR(20) DEFAULT 'dynasty'
  CHECK (league_type IN ('dynasty', 'redraft', 'keeper'))
```

**Fix:** Separated column creation from constraint:
1. Add column without constraint
2. Add CHECK constraint separately in DO block
3. Check if constraint exists before adding (idempotent)

**Deployment:** Pushed 12:03 AM, server healthy 12:05 AM

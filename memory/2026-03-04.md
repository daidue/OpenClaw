### Comp Generation Algorithm

**Feature Vector Design:**
```python
player_vector = {
  'draft_round': 1,  # 1-7
  'draft_pick': 20,
  'college_dominator': 0.32,  # Market share at college
  'recruit_stars': 5,  # 2-5 stars
  'combine_speed': 4.44,  # 40-yard dash
  'year_2_target_share': 0.40,
  'year_2_yprr': 2.14,
  'year_2_age': 22,
  'team_pass_rate': 0.62
}
```

**Similarity Scoring (Weighted Euclidean Distance):**
- Draft round: 20% weight
- Year 2 target share: 25% weight
- Year 2 YPRR: 15% weight
- College dominator: 15% weight
- Recruit stars: 10% weight
- Year 2 age: 10% weight
- Combine speed: 5% weight

**Output:** Top 10 comps with similarity scores 0-1

**Example:** JSN → Justin Jefferson (0.92), CeeDee Lamb (0.89), Ja'Marr Chase (0.87)

### Root Cause Identified: Sports-Reference Anti-Bot Redirect Carousel

**NOT a URL construction bug**
**NOT a search failure**
**NOT a match scoring issue**

**THE BUG:** Sports-Reference implements JavaScript anti-bot redirect carousel

**How it works:**
1. Browser navigates to requested player page
2. Page loads initially
3. JavaScript immediately redirects to random player page (Ozzie Newsome, Adrian Peterson, etc.)
4. Keeps cycling through random pages every ~2 seconds

**What happened in scraper:**
1. `cdp_navigate()` sends navigation command
2. `time.sleep(3)` waits for rendering
3. By time DOM is read, carousel already redirected to wrong player
4. `match_score()` correctly rejects wrong name (e.g., "Cadillac Williams" vs "Max Borghi" = score 30)
5. Scraper marks as "not found"

**Evidence from logs:**
```
[15/3070] Trey Lance → Low confidence: 'Trey Lance' vs 'Ozzie Newsome' (score:44)
[37/3070] Cadillac Williams → Low confidence: 'Cadillac Williams' vs 'Max Borghi' (score:30)
[65/3070] Ted Ginn Jr. → Low confidence: 'Ted Ginn Jr.' vs 'Ozzie Newsome' (score:40)
```

**Pattern:** Streaks of 50, 49, 41, 38, 36, 31 consecutive failures — consistent with carousel triggering for extended periods

### Code Fix Applied

**File:** `scrape_cfb_stats.py` (already updated by debug agent)

**Changes:**

1. **New function: `cdp_navigate_stable()`**
   - Navigates and verifies final URL matches target
   - If redirect carousel detected, logs and retries with backoff
   - Max 2 attempts
   - Returns success/failure boolean

2. **Updated: `try_direct_url()`**
   - Uses `cdp_navigate_stable()` instead of `cdp_navigate()`
   - Now tries both `-1` and `-2` URL suffixes (some players use `-2`)

3. **Updated: `search_cfb_player()`**
   - Uses `cdp_navigate_stable()` for search page navigation

4. **Updated: `scrape_player_stats()`**
   - Uses `cdp_navigate_stable()` for player page navigation
